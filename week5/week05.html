Week 5 Notes

Strict mode: “use strict” – will improve clarity and speed Lint tools check for sloppiness and conventions as well.

Feature detection checks to see if the browser supports this. Modernizer is one library.

Jest, from FB, helps with testing Use alert() to create breakpoints (kind of). Use debugger for real ones.

Use console.log() to print info.

Create error objects with: Const error = new Error(“Error Message”);

Throw error with: throw “message” or throw new Error(message).

Try/catch/finally – same as Java.

Test driven development:

  1. Write tests (that initially fail).
  2. Write code to pass the tests.
  3. Refactor the code.
  4. Test refactored code.
  5. Write more tests for new features.

Jest

Install Jest in command line: npm install -g jest

Run Jest: go to folder and run jest -c {}
